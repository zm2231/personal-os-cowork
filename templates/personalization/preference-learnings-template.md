## Usage Instructions

### For Learning Extraction

**From interaction-history.md:**
1. Analyze recent interactions (last 7-30 days)
2. Identify patterns in user feedback
3. Detect consistent preferences
4. Calculate confidence based on:
   - Evidence count (more = higher confidence)
   - Consistency (less variation = higher confidence)
   - Time span (recent + sustained = higher confidence)
5. Create new learning entry
6. Update personalization-profile.md if confidence >= 70%

### For Confidence Calculation

**Confidence Factors:**
- **Evidence Count:** 10+ interactions = +30%, 5-10 = +20%, 1-5 = +10%
- **Consistency:** 90%+ consistent = +30%, 70-90% = +20%, <70% = +10%
- **Recency:** Last 7 days = +20%, Last 30 days = +10%, Older = 0%
- **Source Quality:** Explicit user feedback = +10%, Pattern inference = 0%

**Maximum Confidence:** 100%
**Minimum Confidence for Application:** 70%
**Minimum Confidence for Reporting:** 50%

### For Learning Updates

**Update learning when:**
1. New evidence contradicts existing learning
2. Confidence threshold crossed (70% for application)
3. User explicitly changes preference
4. Pattern strength changes significantly

**Update process:**
1. Mark old learning as superseded
2. Create new learning entry with updated value
3. Update personalization-profile.md
4. Record evolution in learning evolution section

### For Learning Review

**Weekly:**
1. Review new learnings (last 7 days)
2. Identify learnings ready for application (confidence >= 70%)
3. Update personalization-profile.md
4. Report key learnings to user

**Monthly:**
1. Review all learnings
2. Deprecate learnings with no recent evidence
3. Analyze learning evolution trends
4. Generate monthly insights report

## Notes

- **Learning Source:** Primary source is interaction-history.md
- **Confidence:** Only apply to personalization-profile.md when confidence >= 70%
- **Evolution:** Track all changes to understand user preference evolution
- **Deprecation:** Archive learnings that haven't been used in 90+ days
- **Supersession:** Never delete learnings, always supersede with new entries
