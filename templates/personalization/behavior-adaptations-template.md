## Usage Instructions

### For Applying Adaptations

**From preference-learnings.md:**
1. When a learning reaches confidence >= 70%, create adaptation
2. Document behavior change (before/after)
3. Apply to affected skills (all skills or specific ones)
4. Record in behavior-adaptations.md
5. Set review date (7-30 days out)
6. Monitor impact via interaction-history.md

### For Monitoring Impact

**Track metrics:**
- Acceptance rate (before/after)
- Correction count (before/after)
- User satisfaction signals
- Performance metrics (time, iterations)

**Review schedule:**
- Daily: Quick check for obvious issues
- Weekly: Detailed performance analysis
- Monthly: Comprehensive review and rollback if needed

### For Rolling Back

**Rollback when:**
1. Performance degrades significantly
2. User explicitly requests rollback
3. Conflicting adaptations detected
4. Adaptation confidence drops below 50%

**Rollback process:**
1. Mark adaptation as rolled_back
2. Document reason and lessons learned
3. Revert to previous behavior
4. Update affected skills
5. Notify user if adaptation was user-visible

### For Superseding Adaptations

**Supersede when:**
1. New learning supersedes old learning
2. Better adaptation available
3. Context change requires different adaptation

**Supersede process:**
1. Mark old adaptation as superseded
2. Create new adaptation entry
3. Document relationship between adaptations
3. Apply new adaptation
4. Update affected skills

---

## Notes

- **Application Threshold:** Only apply adaptations when confidence >= 70%
- **Monitoring:** Monitor all adaptations for 7-30 days before considering stable
- **Rollback:** Never hesitate to rollback if performance degrades
- **User Feedback:** Collect explicit user feedback on major adaptations
- **Documentation:** Always document the rationale and expected impact
- **Performance Tracking:** Use interaction-history.md for objective metrics
