# Style Transfer

Transform your writing between different styles—formal, casual, technical, creative—while preserving your core message and meaning.

## When to Use This Skill

Use when you want to:
- Adapt the same content for different audiences
- Rewrite formal content for casual channels
- Make casual writing more professional
- Convert technical content for general audiences
- Create multiple versions of the same message

## Style Transformations I Support

### Level 1: Basic Styles

**Formal ↔ Casual:**
- Expand contractions (don't → do not)
- Add/remove polite phrases
- Adjust vocabulary complexity
- Change greeting/closing
- Modify sentence structure

**Simple ↔ Professional:**
- Add/remove industry terminology
- Adjust sentence complexity
- Include/exclude formal language
- Change tone register
- Adapt to business context

**Clear ↔ Concise:**
- Add explanatory details
- Remove redundant words
- Simplify complex phrasing
- Expand bullet points
- Compress long paragraphs

### Level 2: Context-Aware Styles

**Audience-Specific:**
- Technical audience (add jargon, detail)
- General audience (simplify, explain)
- Executive audience (summary, impact-focused)
- Customer audience (friendly, clear)
- Investor audience (metrics, growth-focused)

**Platform-Specific:**
- Email (professional, structured)
- Slack/Teams (casual, brief)
- Social media (engaging, hashtags)
- Blog post (conversational, detailed)
- Documentation (clear, instructional)

**Industry-Specific:**
- Tech startup (innovative, metrics-driven)
- Corporate (formal, compliance-aware)
- Creative (expressive, unique)
- Academic (precise, cited)
- Healthcare (careful, empathetic)

### Level 3: Advanced Adaptations

**Style Profiles:**
- Personal writing voice
- Brand voice guidelines
- Company communication style
- Team communication norms
- Platform-specific guidelines

**Proactive Suggestions:**

## Best Practices

1. **Meaning preservation:** Core message never changes
2. **Audience awareness:** Style serves the reader
3. **Context matters:** Platform and purpose guide choices
4. **Your voice matters:** Adapt, don't imitate
5. **Iterative refinement:** Ask for feedback

## Examples

### Formal to Casual
```
User: Rewrite this email casually

User: "Dear Mr. Johnson,

I am writing to inform you that your application has been reviewed and approved. Please proceed to the next phase of the process at your earliest convenience.

Sincerely,
Sarah Smith
Project Manager"

Assistant: 💬 Casual Version

"Hey Johnson!

Great news - your application got approved! 🎉

Feel free to move on to the next phase whenever you're ready.

Best,
Sarah"

**Changes:**
- "Dear Mr. Johnson" → "Hey Johnson" (less formal)
- "I am writing to inform you" → "Great news" (conversational)
- "has been reviewed and approved" → "got approved" (casual)
- "at your earliest convenience" → "whenever you're ready" (natural)
- "Sincerely, Sarah Smith, Project Manager" → "Best, Sarah" (simpler)

Core meaning preserved, tone shifted to casual.
```

### Technical to General Audience
```
User: Rewrite this for general audience

User: "Our microservices architecture leverages containerization technology and orchestrates deployment through Kubernetes. The system implements eventual consistency via asynchronous event-driven messaging between services."

Assistant: 📝 General Audience Version

"Our system is built using small, independent services (called microservices). Think of it like a team of specialists, each handling one task.

We use containers to package these services so they run the same everywhere. Kubernetes is our tool that manages all these containers - it's like an automated supervisor.

When different parts of our system need to talk to each other, they send messages. This happens in the background, so if one part is busy, the others keep working smoothly.

In simple terms: Our system is flexible, reliable, and can handle lots of users at once."

**Changes:**
- "microservices architecture" → "small, independent services" + analogy
- "containerization technology" → "containers to package... run same everywhere"
- "orchestrates deployment" → "manages... automated supervisor"
- "eventual consistency" → "happens in background... keep working smoothly"
- Added analogies and simplified language

Preserved technical accuracy while making it accessible.
```

### Contextual Style Detection (Level 3)
```
User: "The server crashed again. I've tried restarting it three times. Any ideas?"

Assistant: 🔍 Context Detection

PLATFORM: Slack
AUDIENCE: Engineering team
TONE: Frustrated, casual
PRIORITY: High (server crash)

RECOMMENDED STYLE: **Technical & Direct**

"🚨 Server is down again

I've already attempted 3 restarts - no luck. Logs show segmentation fault on request processing.

Need eyes on this ASAP if anyone is free. @oncall"

**Why this style:**
- Slack format: Casual but structured
- Technical: Includes details (segfault, logs)
- Direct: Gets to the point quickly
- Tag oncall: Appropriate escalation

**Alternative if manager asked:**
**Style: Summary & Impact**

"Server Status Update:

- Issue: Unexpected server crash
- Attempts: 3 restarts, no success
- Root cause: Under investigation
- Impact: Users experiencing downtime
- ETA: TBD

I'm actively working on this and will update within the hour."

Would you prefer one style or the other?
```

### Voice Consistency Check (Level 3)
```

User: [New company announcement]

Assistant: 🎯 Brand Voice Analysis

YOUR BRAND VOICE (from 100+ samples):
- Tone: Friendly but professional
- Personality: Approachable, human-first
- Vocabulary: Simple, no jargon
- Structure: Short paragraphs, lots of white space
- Emojis: Used sparingly (1-2 per post)
- Sign-off: Always includes team name

THIS POST COMPARED TO BRAND:
Voice Match Score: 65/100

⚠ DOESN'T MATCH YOUR BRAND:

1. Tone: Too formal ("We are pleased to announce")
   → Your brand: "Exciting news!"

2. Vocabulary: "leverage strategic synergies" (jargon-heavy)
   → Your brand: "work better together"

3. Structure: Long paragraphs (5-6 sentences each)
   → Your brand: 2-3 sentences max

4. Emojis: None
   → Your brand: Usually 1-2 relevant emojis

✅ DOES MATCH:
- Professional yet friendly approach
- Clear value proposition
- Customer-focused

RECOMMENDED REWRITE (matching your brand):

"We're excited to announce something new! 🚀

Our latest update is live, and it makes it easier than ever to [value proposition].

Here's what's new:
• [Benefit 1]
• [Benefit 2]
• [Benefit 3]

Try it out and let us know what you think! 💬

– [Your Company Name]"

Want me to apply this rewrite?
```

## Tips for Success

1. **Know your audience:** Style is reader-focused
2. **Preserve meaning:** Don't change what you're saying
3. **Be authentic:** Adapt, don't fake a style
4. **Use templates:** Save successful style transforms
5. **Get feedback:** Ask if the new style worked

## Common Mistakes

- Changing meaning when changing style
- Over-casualizing professional content
- Adding jargon for technical audiences (explain it!)
- Losing your unique voice
- Not testing style on actual audience
